Pitch: Claude Autonomously Wrote a Root-Level Kernel Exploit in 4 Hours — This Is the Cyber "Warning Shot"

On March 26, FreeBSD published a security advisory for CVE-2026-4747 crediting "Nicholas Carlini using Claude, Anthropic" for discovering a remote kernel code execution vulnerability. Days later, researchers at Calif.io asked Claude to develop an exploit. In ~4 hours of actual compute time, Claude autonomously wrote two working exploits that deliver a root shell on unpatched FreeBSD servers — the OS that powers Netflix CDN, PlayStation, and WhatsApp infrastructure. Claude solved six distinct problems: lab setup, multi-packet shellcode delivery, clean thread termination, offset debugging via De Bruijn patterns, kernel-to-userland process transition, and a hardware breakpoint bug. Both exploits worked on the first try. (Forbes)

Why this matters: This is the clearest real-world "warning shot" yet for autonomous offensive cyber capabilities. The exploit-development barrier — the step from finding a bug to weaponizing it — was the last frontier considered exclusive to human expertise. Claude crossed it autonomously. As Forbes notes, the same researcher has since used Claude to identify 500+ additional high-severity vulnerabilities across other codebases. The cost: a few hundred dollars of compute. This collapses the economics of offensive cyber: a capability once reserved for nation-state teams (think Stuxnet) is now approaching commodity pricing. A story could investigate: Does Anthropic's RSP address autonomous exploit generation? Did Anthropic evaluate this capability before shipping the model? And given that Anthropic's own RSP v3 no longer has pre-deployment gates, what stops the next model from being even more capable at this?

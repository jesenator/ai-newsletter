PITCH: 16 Unsupervised Claudes Built a C Compiler That Compiles Linux—What Happens When They Build Something Dangerous?

Nicholas Carlini, an Anthropic Safeguards researcher, published a detailed account of 16 parallel Claude Opus 4.6 agents autonomously writing a 100,000-line C compiler from scratch—without internet access—that successfully compiles the Linux kernel. $20,000 in API costs, 2,000 sessions, two weeks. The agents coordinated via file locks, resolved merge conflicts independently, and when one crashed, the supervisor agent "dove head first into the exact same problem" and also crashed.

"I haven't yet implemented any other method for communication between agents, nor do I enforce any process for managing high-level goals. I don't use an orchestration agent. Instead, I leave it up to each Claude agent to decide how to act." — Nicholas Carlini

Why this matters: This is the first detailed technical documentation of long-running unsupervised agent swarms from inside a frontier lab. The paper reveals the scaffolding is "bare-bones"—agents coordinate through git repos with no formal orchestration. Carlini explicitly notes the system is a "research prototype" being shared publicly. But the same technique that builds compilers could build offensive cyber tools, and the "lessons learned" section reads like a manual for autonomous AI development. Worth investigating: What safeguards exist for these systems? What happens when the agents are pointed at something other than a compiler?

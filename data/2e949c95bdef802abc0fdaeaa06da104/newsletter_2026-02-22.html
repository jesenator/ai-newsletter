PITCH: Russian Hacker Used Claude Code for Mass Firewall Breach—Found on Attacker's Server

Amazon revealed that a Russian-speaking hacker breached 600+ FortiGate firewalls across 55 countries in 5 weeks using generative AI. But the real story is in the technical details: security researchers found folders named "claude-0" and "claude" on the attacker's misconfigured server containing 200+ files including Claude Code task outputs, session diffs, and cached prompt states.

The hacker submitted full victim network topologies—including IP addresses, hostnames, credentials—to AI services asking for help with lateral movement. Amazon notes the attacker had "low-to-medium skill" but AI "greatly amplified" their capabilities.

Why this matters: This is the first documented case of a threat actor using Claude Code specifically in an active intrusion campaign. Unlike hypothetical warnings about AI-assisted hacking, we now have forensic evidence: the attacker's server exposed their full methodology. This invites scrutiny of Anthropic's acceptable use policies and whether Claude Code's capabilities should be restricted for certain operations. Key questions: Did Anthropic's abuse detection flag this usage? Can frontier AI labs detect when their tools are being used in active cyberattacks?

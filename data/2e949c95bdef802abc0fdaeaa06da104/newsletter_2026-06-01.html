PITCH 2: The "@redhat-cloud-services" npm worm targeted MCP and AI-toolkit packages specifically — and bypassed npm 2FA
StepSecurity Wiz StepSecurity finds 31 compromised packages including @redhat-cloud-services/hcc-pf-mcp, hcc-feo-mcp, hcc-kessel-mcp, and platform-frontend-ai-toolkit. The worm uses stolen npm tokens with the bypass_2fa parameter to self-propagate and explicitly tries to evade Harden-Runner. Four-layer obfuscation (ROT-21 → AES-128-GCM → obfuscator.io → custom B5 cipher).
Why it matters: Second major MCP-class supply-chain hit in two weeks (after BadHost/Starlette). MCP servers hold credentials for every system an agent touches; a worm that targets MCP packages specifically is a warning shot for agentic infrastructure. Press the labs publishing official MCP reference servers (Anthropic, OpenAI) on whether they've audited transitive deps, and ask CISA whether the killed Trump EO clearinghouse would have caught this.
PITCH 3: Hackers used Meta's AI support chatbot to take over Instagram accounts — the agent itself was the vulnerability
404 Media (Koebler) Hackers tell 404 Media they bypassed Meta's account-recovery flow by social-engineering the AI support bot into changing the email on victim accounts — including high-profile ones. Trick: VPN to the account's region, then ask the agent to send a code to an attacker-controlled address. Meta has now patched.
Why it matters: First documented mass-scale account-takeover wave where the AI agent itself was the attack surface, not a model jailbreak. Sits next to the Robinhood-gives-Claude-trading-access story from last week and the BadHost MCP bug. Reportable angle: every Big Tech support function being LLM-replaced is now a credential-reset oracle. Worth asking Meta which Llama variant powers the bot and whether the prompt-injection resistance benchmarks Anthropic publishes apply.
